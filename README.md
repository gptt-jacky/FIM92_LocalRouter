# FIM-92 內網位元控制系統

16-Bit 即時位元控制與監控系統，透過 WebSocket 在區域網路內控制硬體設備。

## 系統架構

```
┌─────────────┐     WebSocket      ┌──────────────┐     WebSocket      ┌─────────────┐
│  瀏覽器 UI   │◄──────────────────►│  Node.js     │◄──────────────────►│  硬體設備    │
│ (控制面板)   │   ws://IP:3000     │  Server      │                    │ (刺針/震動器)│
└─────────────┘                    └──────────────┘                    └─────────────┘
```

- **Server** (`server.js`)：WebSocket 中繼伺服器，負責轉發指令與狀態
- **UI** (`FIM92LOCAL.html`)：16-bit 位元控制面板，支援即時監控
- **啟動器** (`LaunchApp.vbs`)：一鍵啟動腳本，自動偵測 IP 並開啟 Chrome App 模式

## 環境需求

- **OS**：Windows 10/11
- **Node.js**：v16+
- **相依套件**：`ws` (WebSocket library)
- **瀏覽器**：Chrome（App 模式）

## 快速開始

### 方法一：一鍵啟動

雙擊 `LaunchApp.vbs`，會自動：
1. 關閉舊的 Node 程序
2. 啟動 server
3. 偵測本機 IP 並顯示分享連結
4. 以 Chrome App 模式開啟控制面板

### 方法二：手動啟動

```bash
npm install
node server.js
```

開啟瀏覽器前往 `http://localhost:3000`

## 通訊協定

| 指令格式 | 說明 | 範例 |
|----------|------|------|
| `SET_X_Y` | 設定第 X 位元為 Y | `SET_3_1`（BIT3 設為 ON） |
| `BIT_W` | 整體設定為數字 W | `BIT_65535`（全開） |
| `CLS` | 清除所有位元 | `CLS` |
| 數字 | 狀態回報 (0-65535) | `34`（BIT1 + BIT5） |

## 位元定義

| Bit | 值 | 名稱 | 方向 |
|-----|-----|------|------|
| BIT0 | 1 | 刺針本體就緒 | 接收 |
| BIT1 | 2 | 震動器就緒 / LED 恆亮 | 接收 |
| BIT2 | 4 | 震動器中震動 | 接收 |
| BIT3 | 8 | 震動器大震動 | 接收 |
| BIT4 | 16 | 刺針鎖定鍵 | 發送 |
| BIT5 | 32 | 刺針 BCU | 發送 |
| BIT6 | 64 | 刺針板機 | 發送 |
| BIT7 | 128 | 刺針保險 | 發送 |
| BIT8 | 256 | 刺針瞄準模組 | 發送 |
| BIT9 | 512 | IFF 開關 | 發送 |
| BIT10 | 1024 | 後座力準備 | 發送 |
| BIT11-15 | - | 未定義 | - |
